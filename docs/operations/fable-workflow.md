# Fable Workflow

Fable should be used for planning, issue writing, goal writing, and review.

## Best Uses

- Read Notion specs and project notes.
- Break work into small GitHub issues.
- Write goal files for Codex in `docs/goals/`.
- Review implementation strategy before Codex starts.
- Review PRs after Codex finishes.

## Issue and Goal Output

Fable output should land in one of:

- a GitHub issue
- `docs/goals/*.md`

Each output should be specific enough that Codex can implement it without guessing.

## Guidance for Fable

- Prefer small tasks over broad implementation plans.
- Split large features into multiple issues.
- Make out-of-scope items explicit.
- Include stop-and-ask conditions.
- Ask a human before deciding major product or architecture changes.

## Fable Is Not The Merge Authority

Fable can recommend, review, and summarize, but a human should make the final merge decision.
