# Objective

Implement the 広がりマップ screen at `/map`: dishes as nodes on an island-like map where made dishes are claimed territory, makeable/saved dishes are footholds, and unexplored neighbors are dimmed (spec v2.8 draft §4.2, §5.2; decision 002 — 達成感 and 探索感).

# Product Context

Spec: `docs/specs/tonari-gohan-spec-v2.8-draft.md` (§4.2, §5.2)
Decision: `docs/decisions/002-near-complete-mvp-validation.md` (island map is owner-confirmed in scope)
Design tokens: `app/globals.css` (`--tn-*`). No concept image exists for this screen — follow the same warm visual language as Home; SVG aesthetics are free within the tokens.

The map is why users feel progress: cooking a dish claims territory; neighbors hint at what's nearby.

# Scope

1. `data/islandMap.ts` — new static layout:
   - `IslandNode[]` per spec §5.2: `{ dishId, x, y, neighbors }` with normalized 0-100 coordinates
   - Cover ALL base dishes and ALL dish cards. Place related dishes near each other (use baseDishIds / tasteAxis / mainIngredients as a guide when hand-placing). Neighbors = 2-4 nearest related dishes
   - Cluster shape should read as one organic island, not a grid
2. `components/mvp/IslandMap.tsx` — new client component:
   - SVG rendering (viewBox, responsive to 375px width; vertical scroll if taller)
   - Island background: simple organic blob shape(s) using token colors (`--tn-surface-soft` land on `--tn-bg` sea is fine)
   - Node states (spec §4.2):
     - made (`madeAt` set): filled accent color, bold label
     - foothold (base dish selected OR saved): mid tone (accent-soft fill, accent border)
     - unexplored: dimmed (low-opacity gray), still labeled
   - Edges: thin lines between neighbors (subtle)
   - Tap a node → open existing `DishDetailModal` for dish cards; base dishes open a small info popover or are non-interactive (your choice, report it)
   - Legend strip explaining the 3 states: `作った` / `作れる・作ってみたい` / `これから`
   - Header copy: `広がりマップ` + `作った料理が、あなたの島になります。`
   - Progress line: `あなたの島：{made数} / {全ノード数} 品`
3. `app/map/page.tsx` — replace placeholder (keep BottomNav).
4. Analytics: add `open_island_map` to the typed event names in `lib/phase0/analytics.ts` and fire it once when the map screen mounts. No other new events.

# Out of Scope

- Pan/zoom/drag (tap + vertical scroll only)
- Animations beyond simple CSS transitions
- Auto-layout algorithms (hand-tuned static coordinates)
- Map libraries or any new dependency
- return_visit event (next goal)
